Om projektet: Svensk Wordle – Fullstack

Välkommen till min tolkning av det klassiska ordspelet Wordle! Detta projekt är skapat som en del av kursen Javascript - fullstack inom ramen för utbildningen Systemutvecklare - Javascript - Java hos Lernia YH i Piteå.

Målet med uppgiften var att bygga en robust och säker webbapplikation där logik, databashantering och användargränssnitt samspelar sömlöst. Spelet går ut på att gissa ett dolt ord på ett begränsat antal försök.

För att göra det extra utmanande har jag implementerat följande:

Anpassningsbara regler: Man kan själv välja ordlängd och om ordet får innehålla dubbletter av bokstäver.

Fuskskydd: All logik för rättning av ord och tidtagning sker på servern. Det går alltså inte att "tjuvkika" i webbläsarens konsol för att hitta rätt svar. Highscore: När man har knäckt ordet sparas tiden och användarens spel-inställningar i databasen, så att man kan mäta sina krafter mot andra spelare.

Teknikstack

Projektet är byggt med en stadig "Node-kloss" i basen:

Frontend:

React med TypeScript blir ett typsäkert och responsivt gränssnitt.

Backend:

Node.js och Express som serverar sidan, API och den server-side renderade highscore-listan.

Databas:

MongoDB via Mongoose för att lagra resultat och ordlistor.

Rendering:

EJS används för att rendera de delar av applikationen som kräver snabb server-side leverans, som t.ex. highscore-listan.

Tankar bakom koden

Jag har mest försökt att lägga fokus på "Separation of Concerns". Det innebär för mig att backend får sköta den tunga logiken och säkerheten, medan frontend får fokusera på att ge användaren en smidig och trevlig spelupplevelse.

Ordlistan hämtas via ett API och den och logiken för ord finns i separata submoduler för att hålla koden ren och lätt att underhålla.
